# Security

## Сообщить об уязвимости

Не создавайте публичный issue для проблем безопасности.

Опишите проблему приватно (email maintainers или Security Advisory на GitHub, когда репозиторий будет опубликован).

## Секреты

- Никогда не коммитьте `.env` с реальными API-ключами
- После утечки ключа — **сразу ротируйте** его у провайдера
- Режим **YOLO** выполняет команды и записи **без подтверждения** — используйте только в доверенных проектах
